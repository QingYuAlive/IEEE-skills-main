# IEEE Evidence Table

Every proposal claim must map to a formal object and an empirical/theoretical defense.

| Claim | Formal object | Evidence/source | OOD protocol | Required ablation | Metric/statistic | Visualization | Complexity evidence | Risk | Status |
|---|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |  | formalized / evidence-backed / ablation-needed / theory-needed / unsupported / forbidden |

## Status Definitions

- `formalized`: symbols/objective/protocol exist, but evidence may be pending.
- `evidence-backed`: data, ablation, or theory supports the claim.
- `ablation-needed`: claim depends on a module/loss not isolated yet.
- `theory-needed`: theorem/convergence/bound is claimed but incomplete.
- `unsupported`: claim lacks formal or empirical support.
- `forbidden`: claim contradicts the current protocol or evidence.
