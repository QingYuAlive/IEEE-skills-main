# Section: Related Work

## Default Structure

`technical obstacle -> representative method family -> formal limitation -> connection to this paper`

## Drafting Rules

- Group by technical mechanism, not by publication year or author order.
- End each subsection with the specific limitation that the current paper addresses.
- Tie limitations to formal objects: domain shift, objective mismatch, pair construction, invariant assumption, optimization cost, missing OOD protocol, or deployment constraint.
- Cite primary sources for the method being discussed.
- Do not use related work to inflate novelty. Use it to define the exact unsolved technical boundary.

## Common Groups For IEEE ML Papers

1. Signal or time-series representation.
2. Domain generalization, domain adaptation, and robust training.
3. Contrastive, invariant, adversarial, and distributionally robust learning.
4. Multimodal fusion, reference-guided learning, or two-stage optimization.
5. Complexity, latency, or deployment-aware learning.
