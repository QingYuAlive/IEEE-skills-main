# Introduction Example: Task First, Then Application

Use this pattern only when the task is niche enough that the reader needs a formal definition before application context.

```text
Cross-force sEMG gesture recognition estimates the gesture label `y in \mathcal{Y}` from a multichannel window `X in R^{C x T}` when training and test windows come from different force domains. This setting is central to myoelectric control because a deployed decoder must preserve class discrimination when muscle activation intensity changes across daily use.
```

The first sentence defines input, output, and domain shift. The second sentence gives application relevance without replacing the formal task.
