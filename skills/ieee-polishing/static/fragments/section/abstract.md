# Section: Abstract

## IEEE Polishing Contract

Polish abstracts into this order:

```text
technical task and limitation -> mathematical mechanism -> algorithmic role -> protocol-bound evidence -> bounded implication
```

## Polishing Priorities

- Replace broad motivation with the formal task.
- Preserve no-equation abstract style while keeping precise technical nouns.
- Bind performance claims to dataset, split, metric, and baseline when the user supplies them.
- Remove unsupported robustness, generalization, or state-of-the-art wording.
- End with a bounded implication, not an expansive promise.

## Formalization Flags

Flag the abstract when it names a method but no objective, loss, alignment rule, or protocol is available in the manuscript context.
