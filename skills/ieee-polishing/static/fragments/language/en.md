# Language: English Source

## IEEE Sentence Rules

- Preserve mathematical density when a definition, objective, or protocol requires it.
- Keep one main technical proposition per sentence unless a formal definition needs controlled compound structure.
- Split sentences that mix task definition, method, result, and implication.
- Do not shorten sentences by deleting variables, domains, coefficients, or protocol qualifiers.
- Avoid em dashes as prose punctuation unless the user explicitly requests them.
- Prefer precise verbs over decorative connectives.
- Keep inline variables in `$...$`.

## Paragraph Rules

- Each paragraph should have one reviewer-facing job: definition, gap, method, protocol, comparison, ablation, interpretation, or limitation.
- Put evidence near the claim it supports.
- Start result paragraphs with protocol or comparison, not with table labels.
- End interpretive paragraphs with a boundary when the claim could overreach.

## Common Repairs

- Replace vague pronouns with canonical terminology when ambiguity affects meaning.
- Replace informal synonyms with the Notation Ledger term.
- Move hedging from direct result sentences to interpretation sentences.
- Preserve author-supplied metric values and statistical wording exactly unless correcting a requested typo.
