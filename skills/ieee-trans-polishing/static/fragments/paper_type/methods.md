# Paper Type: Methods

## IEEE Reader Questions

- Is the method mathematically defined?
- Is it reproducible?
- Is it better under a fair comparison?
- Is the added complexity justified?

## Polishing Priorities

- Convert workflow descriptions into notation, mappings, objectives, and algorithms.
- Require matched baselines and identical splits for performance claims.
- Preserve derivations, assumptions, hyperparameters, and implementation details.
- Include ablation language for each proposed component.
- Include cost language when efficiency, deployment, or real-time use is claimed.

## Argument Order

```text
problem setting -> operator design -> objective -> training algorithm -> inference path -> experiments -> complexity -> limitation
```
