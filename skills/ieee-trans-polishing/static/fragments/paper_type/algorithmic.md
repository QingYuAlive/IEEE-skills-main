# Paper Type: Algorithmic Or System

## IEEE Reader Questions

- What computational object is introduced?
- What objective, constraint, or update rule makes it different?
- How is it evaluated under matched protocols?
- Which ablations isolate its contribution?
- What is the cost at training and inference?

## Polishing Priorities

- Preserve formal notation and algorithmic steps.
- Replace module-list prose with equation-aware wording.
- Bind performance claims to baselines, splits, and metrics.
- Flag missing complexity if deployment, efficiency, or real-time claims appear.
- Make stage-wise training and inference paths explicit.

## Argument Order

```text
formal task -> algorithmic mechanism -> objective -> training/inference -> comparison -> ablation -> complexity -> limitation
```
