# Paper Type: Empirical Research

## IEEE Reader Questions

- What is the formal task?
- Which source and target conditions are tested?
- Which mechanism is claimed to improve performance?
- Which metrics, baselines, ablations, and limitations support the claim?

## Polishing Priorities

- Replace general motivation with technical task definition.
- Bind every empirical claim to dataset, split, metric, baseline, and checkpoint rule when available.
- Preserve negative or mixed results when they define the boundary.
- Treat visualizations as diagnostic support, not proof.

## Argument Order

```text
task -> shift or difficulty -> mechanism -> protocol -> result -> ablation -> limitation
```
