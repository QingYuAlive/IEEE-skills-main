# Section: Results

## IEEE Polishing Contract

Each Results paragraph should follow:

```text
protocol -> comparison -> measured effect -> mechanism or ablation interpretation -> boundary
```

## Polishing Priorities

- Replace `Table I shows...` with a claim-first sentence.
- State dataset, split, metric, baseline, and checkpoint rule when available.
- Keep mechanism interpretation short and tied to ablation or representation metrics.
- Do not let visualizations carry unquantified claims.
- Report negative, mixed, or weak-domain results without hiding them.

## Formalization Flags

Flag claims that say robust, invariant, generalizable, necessary, or significant without the required protocol, ablation, or statistical support.
